**Persona**: QA (orchestrator-authored, not a subagent — see scope note below).
**Task**: Validate the Phase 2 vertical slice (Anansi vs Anansi-bot mirror match) across the three bot difficulties. The brief asked for 9 playtest matches (3 difficulties × 3 runs). This writeup documents what was actually verified, what was substituted, and what still needs human confirmation.

## Scope note — what "playtest" means in this environment

The orchestrator is a headless Claude Code session. Real playtest means a human clicking through a 3D browser game and forming qualitative judgments about *feel* — hitstop weight, input-to-action latency perception, camera composition, audio immersion, tutorial clarity. None of that is available here. What *is* available:

1. **Determinism testing**: `tests/unit/aiBot.test.ts` (T-105) asserts identical InputFrame streams from the bot given identical snapshot sequences + seed. 120-tick replay test passes; the bots are truly deterministic, satisfying ADR-0006.
2. **Automated Playwright smoke** (`tests/e2e/smoke.spec.ts`): full menu → god-select → match → pause → resume loop, HUD elements present, no console errors, settings overlay interactive.
3. **Static balance analysis**: T-100's Anansi kit doc has TTK projections landing in the §6.4 5–7 s sustained / 2–3 s combo bands against 320 / 380 / 420 HP targets. No number outside band.
4. **Bot win-rate hypothesis**: T-105 estimates Easy 15–25%, Normal 45–55%, Hard 65–80% vs median human, derived from aim-noise variance × reaction-ms × ability-usage-policy. Not empirically validated yet.

The 9-match empirical playtest remains **the user's job** via the live Cloudflare deploy — this writeup gives them the test plan.

## User playtest plan (the 9 empirical matches)

Live deploy: **https://pantheon-292.pages.dev**

For each difficulty (easy / normal / hard), run **3 matches**. Total: 9.

### Per-match checklist

- [ ] Match starts with both characters visible (Anansi player + Anansi-bot mirror). **Note**: Phase 2 ships with the bot AI system implemented but NOT yet rendered inside MatchScene — the second Anansi hasn't been instantiated. See "Known Phase 2 gap" below.
- [ ] Input-to-action latency feels < 100 ms on player controls (DESIGN §4). Press W, character should start moving within 1–2 frames perceptually.
- [ ] Silken Dart fires on LMB / tap-release; aim-joystick on touch rotates the mesh.
- [ ] Mirror Thread cast (RMB / button) triggers — console logs `[action] ability` (action stub from Phase 1 Fix 6 validates the input pipeline).
- [ ] Ultimate (Space / button) triggers — console logs `[action] ultimate`.
- [ ] Dodge (Shift / LT) triggers — console logs `[action] dodge` and the 300 ms dodge roll covers 3.6 m (dashSpeed 12 m/s × 0.3 s).
- [ ] Pause (Escape / Start / Menu button) opens PauseMenu; Resume closes it; Exit returns to main menu.
- [ ] No visible frame drops during 60+ s of continuous play.
- [ ] No console errors in DevTools.

### Per-difficulty qualitative notes (use these to evaluate feel)

- **Easy**: the 500 ms telegraph should be *visible* — bot looks like it's "thinking" before it acts. If you can't tell the easy bot is easier than normal, that's a bug.
- **Normal**: bot should use Mirror Thread on cooldown (once every ~8 s). If you never see it cast, behavior tree is misconfigured.
- **Hard**: bot should *lead* shots ahead of your movement. Run laterally at 5 m/s — Hard bot's projectiles should arrive where you'll be, not where you are. If shots always miss behind you, prediction isn't wiring.
