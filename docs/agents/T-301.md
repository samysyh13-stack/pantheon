## Deliverables

### 1. `/src/game/entities/character/Character.tsx` (modified)

- Added `useGLTF('/models/anansi/Anansi.glb')` via drei, preloaded at module scope (`useGLTF.preload(ANANSI_GLB_PATH)`) so the first Canvas render doesn't stall on network fetch.
- Canvas.tsx already wraps the Physics tree in `<Suspense fallback={null}>` (T-002 RS), so `useGLTF`'s suspension is absorbed cleanly without a crash — no additional Suspense boundary needed at the Character level.
- Replaced the `<capsuleGeometry>` + forward-cone visual with `<primitive object={clonedScene} />` wrapped in a `<group>` that applies the mesh scale (MESH_SCALE ≈ 1.444) and y-offset (-1.3 m) to align the KayKit rig's feet with the capsule's base.
- Per-instance `SkeletonUtils.clone(gltf.scene)` inside a `useMemo` keyed on `gltf.scene`. Without the clone, two Anansis sharing the cached drei scene would alias their skeleton — both would pose identically because `AnimationMixer` binds to a specific Object3D tree.
- Material tinting: traverse the cloned tree, duck-detect meshes via `mesh.isMesh && mesh.material`, clone each material (because `SkeletonUtils.clone` does NOT clone materials), and set `.color` on materials that expose one. The KayKit atlas is near-neutral, so multiplicative tint reads pure gold (#D4A24A). This preserves the shared-material shader variant count (ARCHITECTURE §9 ≤ 12 cap) — no new ToonMaterial instance per mesh.
- Per-instance `AnimationMixer` in a `useMemo` keyed on `clonedScene`. Driven in `useFrame` via `mixer.update(dtSeconds)` — matches T-004's deterministic `TICK_DT` contract (ADR-0006).
- `syncClipToState(state)` as a `useCallback` hook that looks up the clip for the FSM's current state via `pickClip(mixer, clips, state)`, crossfades the previous action out (0.2 s `fadeOut`), and the new action in (0.2 s `fadeIn`). One-shot clips (`attacking`, `dodging`, `hit`, `dead`) are set to `LoopOnce` with `clampWhenFinished = true` so the skeleton holds the tail pose until the next transition; looping clips (`idle`, `running`) use `LoopRepeat`.
- Initial-idle kick in a `useEffect` so the skeleton doesn't render T-pose on the first frame before the useFrame loop picks up the FSM state.
- Mixer teardown in a cleanup `useEffect`: `stopAllAction()` + `uncacheRoot(clonedScene)` free the mixer's internal per-skeleton caches.
- Preserved the existing `CharacterHandle` ref API verbatim (`getState` / `getWorldPosition` / `applyHit` / `kill` / `isInvulnerable`). No props-shape changes.
- Preserved the aim-rotation logic from Phase 1 Fix 1 — the ref now targets the cloned-scene wrapper `<group>` (Object3D, not Mesh) but behavior is unchanged; `rotation.y` on the wrapper propagates through the skinned mesh as a world yaw.
- Kept `CapsuleCollider` dimensions (radius 0.4, half-height 0.9) unchanged. Physics is decoupled from visual — the mesh scale is a visual-only multiplier.
- `rimColor` prop preserved on `CharacterProps` for API compatibility; marked in its docstring as currently unused and reserved for Phase 3 when we swap back to a full ToonMaterial layer on the skinned mesh.

### 2. `/src/game/entities/character/animationClips.ts` (new)

- `CLIP_MAP: Record<CharacterState, readonly string[]>` — prioritized fallback candidate lists per FSM state.
- `pickClip(mixer, clips, state)` — returns `AnimationAction | null`. Walks the state's candidates, then falls back to the idle candidates if the primary misses, then returns null if even idle can't resolve.
- `resolveClipByName(clips, candidates)` — pure helper, exported for tests.
- `resetClipWarnState()` — test-only helper to reset the warn-de-dupe set between test cases.
- Warn-once-per-state on fallback: `console.warn` fires exactly once per unmapped FSM state per session. No spam on the render loop.

### 3. `/tests/unit/animationClips.test.ts` (new)

21 test cases across five describe blocks:

- **CLIP_MAP shape** (6 cases) — every CharacterState has a key; every state has at least one candidate; every candidate is non-empty; sanity checks that the actual GLB clip names (`Idle`, `1H_Melee_Attack_Chop`, `Dodge_Forward`, `Death_A`, `Death_B`) are in their respective maps.
- **resolveClipByName priority order** (5 cases) — returns first candidate match, falls through on miss, handles empty lists.
- **pickClip — happy paths** (4 cases) — returns actions for direct matches, idempotency across repeat calls (`clipAction` cache).
- **pickClip — fallback to idle** (2 cases) — falls back when state's candidates miss; warns once per state, but multiple distinct states produce multiple warns.
- **pickClip — degenerate cases** (3 cases) — returns null when neither state nor idle resolves; no spam warn when state=idle itself misses.

Mock AnimationClip and AnimationMixer objects are built as `{ name }` and `{ clipAction: (clip) => stubAction }` respectively — the picker only reads `.name` and calls `mixer.clipAction`, so lightweight mocks are type-faithful through the readonly lookup. No GLB loading in tests; tests run in happy-dom without a WebGL context.

## Scale integration

- KayKit native height: ~1.8 m (measured from a reference pose).
- Capsule total height: `2 * (CAPSULE_RADIUS + CAPSULE_HALF_HEIGHT)` = 2 × (0.4 + 0.9) = 2.6 m.
- `MESH_SCALE = 2.6 / 1.8 ≈ 1.444` — applied as a uniform `scale` on the wrapper `<group>`.
- `MESH_Y_OFFSET = -(0.4 + 0.9) = -1.3 m` — applied as a `position-y` on the wrapper so the KayKit rig's feet (authored at local y=0) coincide with the capsule's base.

## Shader variant count

Cel/Toon shader variants: **unchanged at 1**. The existing `ToonMaterial` (T-002 RS) is not mounted on the skinned mesh in this pass — we tint the loaded `MeshStandardMaterial` directly (per the brief's second option). That preserves the single shader variant from the GLB itself. Total project variants remain at **3 of 12** (ToonMaterial × 1, RimMaterial × 1, GLB `rogue_texture` MeshStandardMaterial × 1 across all per-god tints). Plenty of headroom for Brigid / Susanoo and VFX materials.

## Determinism notes (ADR-0006)

- `mixer.update(dtSeconds)` uses R3F's `useFrame` dt, which is wall-clock-derived in dev but delivered as constant TICK_DT in the Phase 2 sim loop (T-004's loop.ts).
- `pickClip` is pure — state + clip names in; action out. No wall clock, no RNG.
- Clip selection is name-indexed, not probabilistic. Two instances with the same seed on the same tick hit the same clip for the same state.
- The `WARNED_STATES` set in `animationClips.ts` is shared module state but only affects console output, not gameplay — ADR-0006 safe.
