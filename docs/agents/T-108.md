## Bot AI per-tick cost

T-105 self-reported < 0.2 ms per tick for the bot's update. Behavior-tree traversal is trivial (< 10 BT node visits per tick on Normal). Aim noise uses box-Muller (2 log + 2 trig — ~1 µs). Finite-difference velocity is a Vector3 sub + divide. The BotTicker's per-frame overhead in MatchScene is:
- 2× handle.getWorldPosition() (reads rigidBody.translation()): ~2 µs
- 1× Vector3 sub + divide + copy: ~1 µs
- 1× object construction for BotWorldSnapshot: ~2 µs
- bot.update(snap): ~150 µs (T-105's estimate)

**~155 µs total per frame.** At 60 FPS: 0.93% of frame budget. Safe.

## Recommendations for Phase 3

1. **Real-device benchmarking**. Paste the live URL (https://pantheon-292.pages.dev) into a real iPhone 12 / Pixel 6 in the browser; watch the devtools Performance tab; post the p99 frame time. This unblocks the "measured" gate checkbox.
2. **Shader fold (T-102 flag #1)**. Absorb silken-dart-head + eight-strand-telegraph-ring into existing shaders before Brigid's VFX land.
3. **Rapier WASM lazy-load**. 3.2 MB precache is 75% of the bundle; deferring to first match entry drops the initial load to ~1 MB. Trade-off: 300–500 ms first-match delay for WASM init.
