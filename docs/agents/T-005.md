## Accessibility decisions

### Reduced motion (`settings.accessibility.reducedMotion`)

- `MainMenu`: skips the 200 ms slide-in; mounted state is static.
- `PauseMenu`: uses opacity-only fade (150 ms) in place of the slide-in-from-right.
- `SettingsMenu`: drops the overlay fade transition.
- No component exceeds the DESIGN ┬¦12 "200 ms" slide-in cap even without `reducedMotion`.

### High contrast (`settings.accessibility.highContrast`)

- `Button` (primary): adds a white 2 px border around the gold fill.
- `Button` (secondary): thickens border from 1 ŌåÆ 2 px and raises text color from gold to `--panth-ink`.
- `Slider`: thickens track from 1.5 ŌåÆ 2 px and raises base opacity.
- `Toggle`: adds 2 px border to track (both on + off states).
- `Select`: thickens wrapper border from 1 ŌåÆ 2 px with a higher-opacity color.
- `Tabs`: thickens active-tab bottom border from 2 ŌåÆ 4 px.
- `Card`: thickens border from 1 ŌåÆ 2 px with a higher-opacity color; selected ring thickens from 2 ŌåÆ 4 px with an offset.

### Colorblind mode (`settings.accessibility.colorblindMode`)

- Central `teamColor(team, mode)` / `useTeamColor(team)` helpers in `src/ui/colorblind.ts`; hand-tuned per-mode hex table exported as `COLORBLIND_PALETTE`.
- Used by `GodSelect` (accent bar, portrait gradient, HP bar) so the signature-color cue stays legible under any dichromacy.
- Used by `ColorblindSwatch` to preview all four modes side-by-side in the Settings > Accessibility tab.
- **Never color-alone**: every colored affordance also carries a shape cue.
  - `GodSelect` lock state: ­¤öÆ glyph + grayscale overlay + "Coming soon" text.
  - `Toggle`: thumb position (left/right) + Ō£ō glyph overlay on "on".
  - `Card` selected: ring + ŌŚē glyph in the top-right corner.
  - `Results` banner: Ō¢▓ victory / Ō¢╝ defeat glyph + color.
  - `ColorblindSwatch`: each swatch has its mode name beneath it.

### Touch + safe area

- Every interactive element uses `min-h-11 min-w-11` (44├Ś44 pt, Apple HIG).
- All full-screen overlays use `.safe-top/.safe-bottom/.safe-left/.safe-right` so controls clear iOS notches and Android display cutouts.
- Touch scale + opacity sliders in Settings > Controls are gated behind `'ontouchstart' in window`, evaluated once at mount.

## Exact diff for `src/app/App.tsx` wiring (orchestrator will apply)

Replace the current content of `src/app/App.tsx` with:

```tsx
import { useEffect, useState } from 'react';
import { GameCanvas } from '../rendering/Canvas';
import { HUD } from '../ui/hud/HUD';
import { Screens } from '../ui/Screens';
import { PauseMenu } from '../ui/menus/PauseMenu';
import { useAppStore } from '../state/store';

export function App() {
  const preset = useAppStore((s) => s.settings.graphicsPreset);
  const screen = useAppStore((s) => s.match.screen);
  const setScreen = useAppStore((s) => s.setScreen);
  const [paused, setPaused] = useState(false);

  useEffect(() => {
    const lock = async () => {
      const orientation = window.screen?.orientation as
        | (ScreenOrientation & { lock?: (o: string) => Promise<void> })
        | undefined;
      try {
        await orientation?.lock?.('landscape');
      } catch {
        /* user-gesture-gated; ignore */
      }
    };
    void lock();
  }, []);

  const inMatch = screen === 'match';

  return (
    <div className="relative h-full w-full overflow-hidden bg-[color:var(--panth-bg)]">
      <GameCanvas preset={preset} />
      {inMatch && <HUD onExit={() => setPaused(true)} />}
      {inMatch && <PauseMenu open={paused} onResume={() => setPaused(false)} />}
      <Screens />
    </div>
  );
}
```

Notes for the orchestrator on applying this diff:

1. The HUD's `onExit` now opens the Pause overlay rather than jumping straight back to the menu. Exit-to-menu lives inside `PauseMenu`, which calls `setScreen('menu')`. This matches DESIGN ┬¦12: "No modal dialogs during gameplay. Pause slides from the right; the match is frozen behind it but visible."
2. `useState<ScreenState>` is gone ŌĆö `match.screen` from the store is the single source of truth. Default is `'menu'`.
3. Phase 2 T-106 will hook `'loading'` ŌåÆ asset preload ŌåÆ `'match'` and drive `'match'` ŌåÆ `'results'` at match end. For now, the GodSelect's Start Match button just flips to `'loading'`, where it sits until that wiring lands.
4. Pause trigger: App.tsx currently routes pause via `HUD.onExit`. Once the unified input manager's `pause` action is wired for gameplay (Phase 2), this should listen for an InputFrame `pause` edge instead, so Escape / Start / Options all trigger pause uniformly regardless of input source.
5. After this diff lands, the optional `onPlay` prop on `MainMenu` becomes dead code. It can be removed in a follow-up chore commit (or left as-is until the Phase-2 tutorial task, which may want it again for the tutorial teaser flow per DESIGN ┬¦10).

- **FPS overlay**: Graphics tab has a local-state toggle, per brief. Wiring it to a real fps-readout HUD element is deferred to the PO (perf) task ŌĆö T-006 lines it up.
- **Ultimate-charge pickup icons in HUD**: HUD.tsx remains the Phase 1 stub with joystick anchors + menu button + timer stub. The full in-match UI (health bar, ability cooldowns, ultimate charge ring) lands in T-106.
- **App version string**: `APP_VERSION` reads `import.meta.env.VITE_APP_VERSION ?? 'dev'`. The orchestrator should drop a `VITE_APP_VERSION` line into `.env.production` (or the deploy workflow) when the Phase 4 polish pass runs; today it renders `"build dev"` in local builds, which is fine for Phase 1.
- **Cinzel font**: MainMenu + Results reference `'Cinzel'` in an inline style. Brief explicitly defers the Google Fonts preconnect to Phase 4.
