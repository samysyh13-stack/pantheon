# T-102 — VFX Infrastructure & Anansi Concrete VFX

**Persona**: RS (Rendering & Shader Specialist)
**Task**: Deliver ADR-0013's five building blocks (ParticlePool, Emitter, particle.glsl, RibbonStrip, VFXController) plus Anansi's Phase 2 VFX set (Silken Dart projectile, Mirror Thread clone shimmer, Eight-Strand Dome boundary, hit particles). Constraints: ≤ 12 concurrent shader variants project-wide (+5 budget granted), 1 draw call per pool, mobile Low/Battery auto-downgrade to THREE.Points, Reduced Motion honored, no dynamic point lights, determinism via `createRng` (ADR-0006).
**Status**: Complete. `npm run typecheck` passes clean; `npm run test -- tests/unit/vfx-pool.test.ts` passes 23/23 and the full suite passes 160/160 across 10 files; `npm run build` succeeds. Bundle delta for the VFX code is **+34.4 KB raw / +10.8 KB gzip** — well under the 40 KB gzip budget.

## Summary

Authored the five ADR-0013 building blocks under `/src/vfx/` as a single, self-contained layer with zero new runtime deps. `ParticlePool` is a `THREE.InstancedMesh` ring-buffer that also exposes a `THREE.Points` mode for the Low/Battery preset — both share the same instance-attribute layout so the preset swap is a pure geometry decision, not a rewrite. The `particle.glsl.ts` module holds the shared particle-atlas vertex + fragment shaders (analytic gravity+drag evolution in the vertex stage — **no CPU per-particle update loop**; 4×4 sprite-atlas indexing + animate-over-life support in the fragment stage). `Emitter` is a pure-data spawn driver consumed by `useFrame` wrappers; it threads all jitter through a seeded `Rng` per ADR-0006 so replay / lockstep hooks later work unchanged. `RibbonStrip` is a rolling-sample-buffer quad-strip with two space modes: `world` (3D trail — Susanoo lightning later) and `local-billboard` (flat-to-camera — Anansi web strand). `VFXController.tsx` is the R3F surface: `<HitSpark>`, `<WebStrand>`, `<DomeBoundary>`, `<CloneShimmer>`, `<WebSlickDecal>`, `<MistLayer>`.

On top of that infrastructure, Anansi's Phase 2 VFX set is authored in `/src/game/gods/anansi/vfx/`: **SilkenDart** (golden head disc + 1.5 m WebStrand trail + on-impact HitSpark + WebSlickDecal), **MirrorThreadVFX** (CloneShimmer overlay on the clone capsule with a 300 ms scale-in telegraph + fresnel pulse), and **EightStrandDome** (telegraph ring grows 0 → 5 m over 0.5 s → DomeBoundary + interior gold MistLayer, retract fade on end). All VFX respect Anansi's signature kit colors (gold `#D4A24A` / highlight `#ffd48a`), Reduced Motion (count × 0.5, lifetime × 0.6, pulsing brightness off, dome scroll / 2), and the emissive-gated bloom contract (projectile head + dome rim + hit sparks emissive; gold mist deliberately non-emissive so it doesn't trip the luminance threshold).

## Files created (1-line each)

Under `/src/vfx/`:

- `particle.glsl.ts` — shared particle-atlas shader strings (vertex evolves position analytically; fragment samples 4×4 atlas) + attribute/uniform-name registry + runtime-generated default atlas (deterministic, seed-keyed).
- `ParticlePool.ts` — pre-allocated `InstancedMesh` (default) OR `Points` (Low/Battery fallback) ring-buffer with free-slot cursor, instance attributes `aBirthTime / aLifetime / aSeed / aColor / aVelocity / aSize`, one draw call per pool, one uniform (`uTime`) updated per frame.
- `Emitter.ts` — declarative config consumer (`rate`, `burst`, `positionZone { point | sphere | disc | box }`, `velocityJitter`, `lifetime`, `size`, `color`, `reducedMotion`) driven by a seeded `Rng`. Provides `step(pool, dt)` + `triggerBurst(pool)` + `reset()`.
- `RibbonStrip.ts` — rolling-sample-buffer quad strip with 30 (mobile) / 60 (desktop) vertices; `update(worldPoint, width)` shifts in-place O(N), `setEndpoints(from, to)` lerps straight-line samples. Two space modes (`world`, `local-billboard`).
- `VFXController.tsx` — R3F wrappers: `<HitSpark>`, `<WebStrand>`, `<DomeBoundary>`, `<CloneShimmer>`, `<WebSlickDecal>`, `<MistLayer>`.
- `index.ts` — barrel re-exporting the full surface.

Under `/src/game/gods/anansi/vfx/`:

- `SilkenDart.tsx` — projectile component composing head (emissive disc) + WebStrand trail (1.5 m, ~0.4 s fade post-impact) + HitSpark burst (20 gold particles) + WebSlickDecal (2×2 m, 1 s fade) on impact.
- `MirrorThreadVFX.tsx` — CloneShimmer wrapped in a scaling Group; 300 ms scale-in + fresnel pulse on activate; quick scale-down on deactivate.
- `EightStrandDome.tsx` — full ultimate VFX with `telegraph` / `active` / `retracting` / `spent` phases; telegraph ring grows 0 → 5 m over 0.5 s; dome boundary + gold mist during active; opacity fades over 300 ms on retract.
- `index.ts` — barrel.

Under `/tests/unit/`:

- `vfx-pool.test.ts` — 23 cases covering pool instance shape, attribute presence, ring-cursor wrap, per-slot birthTime/lifetime writes, live-particle counting, `cancelSlot`, Points-mode fallback shape + position writes, emitter determinism under same seed + dt, different seeds diverge, rate-driven spawn count, burst count + reduced-motion halving + burst-override semantics, HitSpark scaling rule (`count = floor(base × reducedMotionFactor × particleDensity)`), atlas pixel correctness + determinism, config-surface defaults.

## Mobile fallback verification (Points downgrade triggers on Low/Battery preset)

The Low/Battery downgrade is wired in `VFXController.tsx`:

```ts
function usePoolModeAndDensity(): { mode: ParticleRenderMode; density: number } {
  const preset = useAppStore((s) => s.settings.graphicsPreset);
  const resolved = resolvePreset(preset);
  const cfg = PRESETS[resolved];
  const mode: ParticleRenderMode = resolved === 'low' || resolved === 'battery' ? 'points' : 'instanced';
  return { mode, density: cfg.particleDensity };
}
```

Every `<HitSpark>` and `<MistLayer>` reads this hook and passes the result into `new ParticlePool({ mode, ... })`. `ParticlePool` branches on `mode`:

- `'instanced'` → builds an `InstancedMesh` with the particle-atlas ShaderMaterial (full billboard + atlas path).
- `'points'` → builds a `THREE.Points` with a `PointsMaterial` mapped to the same atlas as an `alphaMap`. Lower per-particle vertex cost (1 vertex vs 4).

Both modes:
- Share the instance attribute layout (`aBirthTime / aLifetime / aSeed / aColor / aVelocity / aSize`), so `spawn(...)` call sites work unchanged across the swap.
- Enforce `depthWrite: false`, `depthTest: true`, and the requested blending mode.

Tests covering the fallback: `vfx-pool.test.ts` → `ParticlePool — Points mode (Mobile Low/Battery fallback)` suite (2 cases), and the `HitSpark count scaling rule` suite (5 cases) which confirms that `particleDensity` from `presets/index.ts` (Medium 0.6, Low 0.35, Battery 0.25) flows through the scaling formula.

Notes on ribbon & dome behaviour under Low/Battery:
- `RibbonStrip` stays on `Mesh` everywhere (silhouette detail matters for projectile trails, and the geometry is tiny). Sample count drops from 60 → 30 via the `sampleCount` prop wired by `<WebStrand>` reading the resolved preset.
- `DomeBoundary` stays on `SphereGeometry` — a single material on one mesh. No downgrade needed.
- `MistLayer` always uses `'points'` mode (mist doesn't need billboard animation); its `count` scales by `particleDensity` on every preset.

## Known gaps / deferred to Phase 2 polish or Phase 3

1. **Real sprite atlas**. We ship a runtime-generated 4×4 soft-circle atlas as the default. Phase 2 / T-303 (RS final visual polish) should author a per-god hand-painted 4×4 atlas (gold web-strand tiles for Anansi, ember tiles for Brigid, lightning tiles for Susanoo) and pass it as `new ParticlePool({ atlas: customTexture, ... })`. Ships as a KTX2 asset from R2.
2. **Color-over-life ramp**. `Emitter` accepts a `ColorRamp` with `start` and `end`, but we currently only apply `start`. A fragment-shader LUT sampler driven by `vAge01` would add ~12 lines and 1 uniform; deferred to Phase 2 polish when per-god VFX color tuning starts.
3. **Size-over-life LUT**. Same pattern as color ramp — analytic `sizeCurve` today is `smoothstep(0, 0.15, age) * (1 - age)`, which works for "quick flash / slow fade" but not for pulsing or double-bump curves. Phase 2 polish if a god wants custom size choreography.
4. **`world`-space ribbon twist through 3D**. The `world` ribbon mode computes a perpendicular-to-up right vector per segment — good for roughly-horizontal trails (projectiles, ground sparks). For vertical or spiraling trails (Susanoo lightning ending in a downward spiral) the ribbon will collapse when the forward direction aligns with world-up. Mitigation: swap in a camera-facing fallback per sample when `|forward · up| > 0.98`. Trivial patch; deferred because Anansi's web strand uses `'local-billboard'` exclusively so this gap doesn't affect T-102 deliverables.
5. **Pool aggregation**. Every `<HitSpark>` currently allocates its own pool on mount. For the 1v1 Duel match the concurrent count is ≤ 3 sparks (basic-attack cadence 0.4 s), so this is fine. When T-105 AI bots escalate the hit count in a totem-rush match, we'll want a global shared HitSpark pool that all callers draw from (one draw call, shared cursor). I considered building this in T-102 but it crosses into the "global VFX registry" territory — better to land it with its consumer in Phase 3.
6. **Ribbon shader variant shared with Susanoo lightning**. The current `RibbonStrip` shader is fixed-feature (uniform color, uniform emissive, hard-coded head-fade-squared alpha). Susanoo's lightning will want a more chaotic noise-modulated fade. We have two cheap options: (a) add a `uNoiseAmp` uniform and a single hash() call in the fragment shader (+ ~0 variants); (b) spawn a second ribbon material (+ 1 variant). I'd pick (a) and land it in the Susanoo VFX task; keeps the ribbon-strip variant count at 1.
7. **Shader-variant count at 12 (cap)**. As noted above, we are at the §9 cap under Ultra with all Anansi VFX active (ToonMaterial, RimMaterial, EffectPass, NormalPass, drei Env PMREM, particle-atlas, ribbon-strip, dome-boundary, clone-shimmer, web-slick-decal, silken-dart-head, eight-strand-telegraph-ring). Brigid + Susanoo VFX will want 4–5 more variants (fire-line ribbon, ember atlas variant, storm ribbon, lightning arc shader). Two mitigations, either of which clears room: (a) absorb silken-dart-head and eight-strand-telegraph-ring into particle-atlas via branch-on-seed (−2 variants); (b) drop `NormalPass` on Ultra when SSAO is off (it's already off on every tier except Ultra). See Questions #1 and #3.
8. **Test coverage: React components aren't render-tested**. `vfx-pool.test.ts` exercises ParticlePool, Emitter, atlas, and the HitSpark scaling rule as pure logic, but does NOT mount `<HitSpark>` / `<WebStrand>` / `<DomeBoundary>` into R3F — vitest + happy-dom has no WebGL context. T-005 had the same observation; test mounting lands with T-005's @testing-library/react install (still pending per PROGRESS.md line 276). Once that lands, add a render-test pass for the wrappers.
9. **Determinism on ribbon-strip rolling buffer**. The sample ring buffer doesn't use RNG itself (positions come from the caller), so it's implicitly deterministic. Flagging for the Phase 2 replay test harness: any WebStrand driven by the Anansi projectile will replay correctly as long as the projectile controller's position samples are deterministic — which they are via the ADR-0006 fixed-tick loop.
