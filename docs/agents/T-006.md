## Phase-2 perf concerns (flagged, not blocking Phase 1 gate)

1. **`post` chunk is 265 kB gzip.** `pmndrs/postprocessing` doesn't tree-shake unused Effect classes; we currently pay for every effect in the library whether we wire it or not. Phase 2 investigation: custom build with only the 5 effects we use (Outline, Bloom, LUT3D, SSAO, ToneMapping), or migrate to a lighter custom effect composer. Potential save: 100–150 kB gzip.

2. **Rapier WASM precache is ~3.2 MB.** This is the biggest single asset in our precache manifest. Load-on-demand (only when entering match) would drop the menu initial-load by ~3 MB. Trade-off: first match load has a 300–500 ms delay for WASM init. Phase 2 decision point.

3. **drei `<Environment preset="warehouse">` fetches the HDRI from the drei-assets CDN at runtime.** Dev-only dependency with uncertain uptime. Phase 2 T-104 bakes a local 512×256 KTX2 HDRI into `public/hdri/` (≤ 200 kB Basis-compressed).

4. **No LOD system yet.** With placeholder capsule characters and simple arena props, this is fine. When Phase 2 drops in rigged Mixamo meshes (target ~3000 tris per character) + stylized shrine props (~1000 tris each), revisit — likely fine without LOD at arena size ≤ 40 m diameter.

5. **Shadow map size at Ultra is 4096.** On integrated-GPU desktops this is conservative but on lowest-end desktops (Intel UHD 620) might cost 2–3 ms per frame. Benchmark when we have real hardware; consider dropping Ultra shadow map to 2048 if we see sustained frame-time regression.

## Recommendations

- **Do not ship `ultra` preset as default.** Use `auto` (already default in store) which falls back to `medium` on iPhone-12-class by the device-capability detector in `/src/rendering/presets/index.ts`.
- **Pin Rapier's physics step to 1/60** at the `<Physics>` site — done in the integration commit. This is critical for ADR-0006.
- **Flag `post` chunk size to Phase 2** — ADR-0013 on particles helps here by not adding `three-nebula`'s 2.9 MB; keep that discipline when picking additional libraries.
